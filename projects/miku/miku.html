Senbonzakura

I started this quest with a goal in mind: to make an acrylic stand that had both bottom and back lighting. For the design of my stand, I took a snippet from an official drawing of Sakura Miku and traced it in Adobe Illustrator.

As for the rest of the stand, I knew I needed something to hold LED strips, so I CADed a container for all of my parts. The backlight would be taped behind the acrylic, and the bottom light would have a slot where it could rest.

Click here for a link to the Onshape model!

With a prototype of my design in mind, I then set off to use the laser cutter to engrave the aforementioned design onto a piece of pink acrylic before cutting it into the shape I needed. With this piece of acrylic, I used the LED strips to find a color that would shine through the best.

Some simple wiring and soldering later, I was finally able to assemble the first version of the Miku box.

And yet, this was quite boring. It was rainbow and all, but there wasn't really any user interaction. As such, I changed that. In the new version, there's a user interface where you can input a Youtube link and a name for it. Python code will then take that name, search it up, and use the source file of the first image it finds to get its average color.

The code will also progress through the song, finding the dominant frequency in order to determine what color the RGB lights should show, with the center of the color range being the average color from earlier. More specifically, the code writes to two text files (one represents the harmony, and the other represents the percussion) so that when a given song plays, these text files are written to an Arduino, which changes the color of 2 LED strips. Due to my bad code, however, everything needs to be in a folder called New Folder (3) (oops).

Click here to download the code! (display.py is the main file, miku_standard.ino is the rainbow version, miku_run.ino is the audio visualization version)
